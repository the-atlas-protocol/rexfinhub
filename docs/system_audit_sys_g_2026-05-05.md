## Top 3 Critical Bypass Paths

### 1. `--bypass-gate` CLI flag — no access control beyond SSH
Anyone with SSH to VPS can run `python scripts/send_all.py --bundle all --send --bypass-gate` and fire all 7 reports to the full production list. No token check, no recipient diff check, no confirmation prompt. Only barrier is SSH.

### 2. Admin password `<redacted-legacy-value>` is in CLAUDE.md (public on GitHub)
Admin access lets you add recipients to any list and trigger sends. L5 diff check is advisory only — emails Ryu a warning but does not block. Attacker adds 1-3 recipients (below the diff threshold of 3), preflight generates warning, Ryu clicks GO, attacker receives reports.

### 3. `send_email.py` direct call bypasses L3/L4/L5
`scripts/send_email.py` is a fully independent CLI calling `_send_html_digest` directly. Inherits L1/L6/L7/L8 but has no atomic gate lock (L3 — gate stays open if it crashes mid-bundle), no preflight token check (L4), no recipient diff check (L5). `--force` additionally bypasses L2 dedup.

## Top 3 Adversarial Scenarios

### Scenario 1: Admin credential compromise → recipient poisoning
1. Attacker authenticates as admin (password `<redacted-legacy-value>` from CLAUDE.md)
2. Adds their email to `autocall` list (which has 9 external RBC/CAIS recipients)
3. Next preflight generates diff warning (1 added, threshold 3 = warn not fail)
4. Send proceeds; attacker receives all future autocall reports
5. **No alert fires on recipient addition.** Audit log shows the addition only via DB inspection.

### Scenario 2: Token TTL gap — stale GO authorizes future send
The idempotency token writes `valid_for_hours: 4` but `send_all.py` never reads the creation timestamp. If a stale decision file from a prior session matches current token UUID, the GO is honored. No automatic decision-file cleanup means HOLD can be accidentally overridden.

### Scenario 3: Gate timing race — concurrent admin panel send
`send_all.py` opens the gate for 30-90 seconds. During this window, any admin endpoint (`/reports/send-li`, etc.) sees `gate=true` and fires without L2 dedup. Result: duplicate sends to RBC + CAIS external recipients.
